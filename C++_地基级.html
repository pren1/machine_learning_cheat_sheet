双指针以及悬空指针： 这段C++代码涉及动态内存分配、内存初始化、指针操作和内存释放。下面是逐行解释：

  1. char* buffer = new char[8]

    这行代码动态分配了一块内存空间，大小为8个字符。buffer 是一个指向 char 的指针，指向这块新分配的内存的首地址。

  2. memset(buffer, 0, 8)

    memset 是一个在 <cstring> 或 <string.h> 头文件中定义的函数，用于将一块内存中的所有字节都设置为特定的值。在这里，它把 buffer 指向的内存的前8个字节都设置为0。这通常用于初始化或清零内存。

  3. char** ptr = &buffer

    这里定义了一个指向 char 指针的指针，名为 ptr。它被初始化为 buffer 的地址。现在 ptr 指向 buffer，而 buffer 指向分配的内存块。

  4. delete[] buffer;

    最后，这行代码释放了 buffer 指向的内存。由于 buffer 是通过 new[] 操作符分配的，所以这里必须使用 delete[] 操作符来释放它。这是为了防止内存泄漏，即分配的内存在不再需要后应该被释放。

需要注意的是，delete[] buffer; 后，buffer 所指向的内存已经被释放，此时 buffer 变成了悬空指针，也就是说它的值仍然是原来内存块的地址，但那块内存已经不再属于程序。在实际的程序中，应该将 buffer 设置为 nullptr 来避免后续潜在的使用无效指针。同样，ptr 仍然指向 buffer，但是由于 buffer 是悬空的，通过 ptr 间接访问该内存也是未定义行为。

这段代码没有显示出来的一个潜在问题是，ptr 依然保留了指向 buffer 的指针，即便 buffer 已经被释放。在实际程序中，如果再通过 ptr 来访问 buffer 所指向的内存，将会导致未定义行为，可能会引发运行时错误。在释放 buffer 后，应当将 buffer 和所有指向 buffer 的指针都设置为 nullptr，以防止这种情况发生。

有趣的知识：我想提到指针的一个有趣行为。如果一个指针被递增，比如 ptr++;，它所持有的地址并不总是递增一字节。相反，它是按照其类型的大小递增的。因此，一个4字节整型指针0x00000004，如果你键入 ptr++;，它将变成0x00000008。这就是 arr[i] 对于连续的 i 值是如何工作的。

在C++中，函数参数可以按照以下三种类别来传递：

  1. 按值传递 (Pass by value): 这种方式会创建参数的一个副本。在函数内对参数的修改不会影响到原始数据。

    xxxxxxxxxx
    void func(int x) {
        x = x + 10; // 只修改了x的副本
    }

  2. 按引用传递 (Pass by reference): 这种方式允许函数直接操作原始数据，而不是它的副本。在函数内对参数的修改会影响到原始数据。

    xxxxxxxxxx
    void func(int &x) {
        x = x + 10; // 修改了x本身
    }

  3. 按指针传递 (Pass by pointer): 类似于按引用传递，允许函数通过指针操作原始数据。在函数内对参数的修改会影响到原始数据。

    xxxxxxxxxx
    void func(int *x) {
        *x = *x + 10; // 修改了x指向的值
    }

这三种传递参数的方式各有用途和特点，在C++编程中广泛使用。按值传递用于简单的数据类型和不需要修改原始数据的场合。按引用传递和按指针传递则用于需要直接修改传入数据或传递大型数据结构（如类和结构体）以避免不必要的拷贝开销的情况。

Array

数组是一种数据结构，可以在一个名称下存储多个元素。

栈上的数组（Stack）

xxxxxxxxxx
int example[5];

这是在栈上声明的数组。这种数组的生命周期是由程序的作用域决定的，当数组的作用域结束时，它会自动被删除，不需要程序员手动管理内存。

堆上的数组（Heap）

xxxxxxxxxx
int* another = new int[5];

这是在堆上动态分配的数组。在堆上分配内存需要程序员手动管理，包括分配和释放内存。

xxxxxxxxxx
delete[] another;

使用 delete[] 来释放 new 分配的数组内存，避免内存泄露。

使用标准库中的数组

C++ 标准库提供了一个模板类 array，它是一个固定大小的数组。

xxxxxxxxxx
array<int, 5> another;

使用 array 类型的数组可以更方便地管理数组，因为它提供了成员函数 .size() 来获取数组的大小。

xxxxxxxxxx
for (int i = 0; i < another.size(); i++) {
    // 使用 another[i] 访问元素
}

Vector 动态数组

vector 是 C++ 标准模板库的一部分，提供了可以动态扩展的数组。

为什么使用 Vector

与传统的数组相比，vector 提供了许多优点：

  • 动态大小：vector 可以根据需要自动调整大小。

  • 内存管理：vector 会自动管理存储空间，不需要手动分配和释放内存。

  • 易用的接口：vector 提供了许多辅助函数，如 push_back、pop_back、size 等，使得操作更加方便。

示例

xxxxxxxxxx
#include <vector>


std::vector<int> vec; // 声明一个int类型的vector


vec.push_back(10); // 在vector的末尾添加一个元素
vec.push_back(20);


// 使用下标访问元素，和数组类似
int firstElement = vec[0];


// 使用size()函数获取vector的元素个数
size_t numberOfElements = vec.size();


// 遍历vector
for (int i = 0; i < vec.size(); ++i) {
    std::cout << vec[i] << std::endl;
}


// 清空vector
vec.clear();


// 检查vector是否为空
bool isEmpty = vec.empty();

字符串 String

在 C++ 中，字符串通常被视为字符数组。

char 类型

char 类型通常占用 1 字节的内存，可以表示 0 到 255 的整数值，这取决于它是 signed 还是 unsigned。

xxxxxxxxxx
const char* name = "Cherno";

这是一个指向字符常量的指针，"Cherno" 实际上是一个字符数组。

xxxxxxxxxx
char* name = {'C', 'h', '\0'}; // 字符串以 '\0' 结束，表示 NULL。

这里定义了一个字符数组，以空字符 \0 结尾，这是字符串的标准表示方式。

如果没有使用 new 关键字分配内存，就不应该使用 delete 关键字。

字符串的模板版本

xxxxxxxxxx
#include <string>
std::string name = "CHERNO"; // 实际上是 const char 数组...

这里使用了 C++ 标准库中的 string 类型，它是对常量字符数组的封装。

xxxxxxxxxx
name.size(); // 直接获取字符串长度

size() 函数可以直接返回字符串的长度。

xxxxxxxxxx
std::string name = std::string("hello") + "there"; // 连接两个 const char 数组的方式

使用 + 运算符可以连接两个字符串，这是一个非常便捷的特性。

xxxxxxxxxx
bool contains = name.find("no") != std::string::npos; // npos 是一个特殊值

find() 函数用于在字符串中查找子字符串，如果没找到，会返回 std::string::npos。

字符串拷贝的性能

字符串拷贝可能很慢，所以当你需要将字符串传递给函数时，确保通过常量引用传递。

xxxxxxxxxx
void print(const std::string& string) {} // 这样传递是快速的

通过使用常量引用传递字符串，可以避免拷贝整个字符串，从而提高效率。

Const 关键字

const 是你对编译器的一个承诺，表明你不会改变变量的值。

指针与 Const

当涉及指针和 const 的组合时，需要注意 const 关键字是出现在星号(*)的前面还是后面，这决定了不可变的是指针指向的数据还是指针本身的值。

1. Const 在星号前面

xxxxxxxxxx
const int* a = new int;
int const* a = new int;

这两种声明方式都表示 a 是一个指向 const int 的指针，你不能通过 a 改变其指向的数据，即使用 *a = 2 是不允许的。

2. Const 在星号后面

xxxxxxxxxx
int* const a = new int;

这里声明的是一个指向 int 的 const 指针，意味着你不能改变指针 a 本身的值，也就是说，a 不能再指向别的地址。

Const 成员函数

在类中，const 成员函数表示该函数不会修改类的任何成员变量。

xxxxxxxxxx
class Entity {
private:
    int m_x;
    mutable int m_y;


public:
    int GetX() const {
        this->m_y = 3; // mutable 允许你在 const 函数中修改参数。
        // 你不能修改类的其他成员参数。
        return this->m_x;
    }
};

在 const 成员函数中，你可以修改用 mutable 关键字标记的成员变量。

Const 引用作为函数参数

xxxxxxxxxx
void PrintEntity(const Entity& e) {
    e.GetX(); // 这要求 GetX 必须是 const 的，因为我们假设输入的 e 是 const。
}

当你将一个对象作为 const 引用传递给函数时，你只能调用该对象的 const 成员函数。

Const 和指针的声明

xxxxxxxxxx
int *m_x, m_y;

这个声明会创建一个指向 int 的指针 m_x，和一个 int 类型的变量 m_y。

Static 关键字

static 关键字在C++中有多种用途，依据它的使用上下文，它的意义会有所不同。

在类外部

全局变量

xxxxxxxxxx
int g_Variable = 5; // 全局变量

这是一个普通的全局变量，可以在包含它的文件及外部文件中被访问（前提是使用了extern关键字）。

extern 关键字

xxxxxxxxxx
extern int e_variable; // 尝试在其他文件中找到这个变量

使用 extern 关键字声明变量时，告诉编译器这个变量在别的文件中定义。

Static 全局变量

xxxxxxxxxx
static int s_Variable = 5; // 本地全局变量, 其他文件不可见

静态全局变量只在定义它的文件内部可见，对其他文件不可见。

静态函数

xxxxxxxxxx
static void function(); // 在其他文件中不可见

静态函数只能在定义它的文件内部被调用，它对其他文件不可见。

当你包含头文件（.h）时，实际上是将头文件的所有内容复制到这里。

在类内部

静态成员变量

xxxxxxxxxx
struct Entity {
    static int x, y;
};


Entity::x = 5; // 这个 x, y 由于是静态的，它们实际上不属于任何Entity的实例...

静态成员变量在所有实例中只有一个版本。

静态方法

静态方法可以不通过类的实例来调用。

xxxxxxxxxx
Entity::Print(); // 静态方法不能访问类中的非静态变量！（当然……）

类提供了带有隐藏指针的函数，但对于静态成员来说，它没有这样的指针。

ENUM 枚举类型

enum 是一种数据类型，它允许程序员为一组值定义别名，这些值实质上都是整数。

为什么 unsigned char 是整数类型？

xxxxxxxxxx
enum Example : unsigned char {
    A, B, C; // A 默认从 0 开始
};

在这个例子中，我们定义了一个 enum 类型 Example，它的底层类型被指定为 unsigned char。unsigned char 是一种整数类型，可以表示0到255的整数值。在 enum 中使用它可以减少存储空间的占用，因为每个枚举值仅需要一个字节。

枚举变量的使用

xxxxxxxxxx
Example m_example = A;

这里 m_example 被声明为 Example 类型，并被初始化为 A，其实际整数值为0。

lvalues 和 rvalues

在C++中，lvalues 指的是定位值，它可以出现在赋值操作的左边。而 rvalues 指的是临时值，它不能被赋值。

lvalues 和 rvalues 的例子

xxxxxxxxxx
int i = 10; // i 是 lvalue, 10 是 rvalue

函数中的 lvalues 和 rvalues

xxxxxxxxxx
void PrintName(string& name) // 无法接受 rvalue
void PrintName(string&& name) // 无法接受 lvalue
void PrintName(const string & name) // 能够接受 rvalue 或 lvalue

在这里，第一个函数 PrintName 只能接受 lvalues 作为参数，因为它接受的是一个引用。 第二个函数使用了右值引用 &&，它专门用来接受 rvalues。 最后一个函数使用了 const 引用，它可以接受 lvalues 或 rvalues 作为参数，因为 const 引用可以绑定到 rvalues。
